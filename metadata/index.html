A Metatadata Application Profile for Data Repository Resources

This work was my final project for IST 681 Metadata, a course offered jointly by the Universities of British Columbia (Canada) and Syracus (United States). My project focused on creating appropriate metadata standards for open source data repository resources in order to improve their discovery, access and retrieval. Data repositories offer an ideal way for researches to publish and share data they create with others. This is essential for ensuring that such data can be preserved and that others can easily find them. Data repositories often do this by archiving submitted data, assigning them a digital object identifier (DOI), and providing a web content that carefully describe each data object including what they are about, how to cite them, or the number of times they are cited or downloaded.

Despite the proliferation of open source repositories for facilitating data exchange, discovery and reuse, not enough attempts have been made to create core metadata standards to aid in their description. Creating such standard could be important for many reasons, the primary being that it would greatly enhance the finding, selection and retrieval of datasets by users based on their specific topics, needs or interests. A related reason could also be that it would enhanced transparency and trust in scientific knowledge production as other scholars would be able to easily discover and work on datasets submitted by their peers, and by so doing, validate and reproduce their findings and thus extend the pace of scientific discovery by building on top of their colleagues' efforts. One likely use case for such a metadata standard could be that a library - most probably academic or a similar research institution - could include metadata records of such repositories in their catalogs for members - researchers, students and similar users - to find, retrieve and resuse.

Thus, given the level of interest in research data exchange and resuse, and the emergence of data repositories as important tools for facilitating these activities, implementing core standards for describing the resources of such repositories has become crucial. This project therefore looked at uncovering the most salient metadata elements for data repository resources that can best enhance their discoverability. I begun with a domain analysis of users and use case scenarios that may be found in the next section.

User Domain Analysis

Although open source repositories for data exchange attract many users, the main ones may be grouped into three categories: users who contribute data to such repositories, other users who access the published or contributed datasets, and universities, libraries and similar institutions that hosts and or maintains such repositories.

The creators or contributors of datasets decide on the title, date, astracts or descriptions, format and similar attributes of the dataset whereas user communities often encompass researchers from academia and sectors such as private business, health, government, or non-profits. These entities perform several activities during interaction with datasets on repositories such as searching, downloading or harvesting the datasets, visualizing and in some cases, even modifying the datasets to suit their needs.

Host institutions or organizations include academic, research, private or government institutions and libraries that create, host and maintain repositories to house contributed datasets. They are often responsible for granting access both to data contributers to submit their data onto the systems and to users to search for and access these datasets. They also provide unique identifiers to submitted objects and create special webpages for hosting contributed datasets. They make rules on how the datasets is to be accessed and used on the repository, and create facilities for users to cite the data they access on the system

Understanding these various entities and how they interact with open source data repository systems is important for uncovering relevant metadata descriptions or elements that can enhance discoverability of objects.

Element Set Specification

The lack of an appropriate and suitable metadata standard for describing data repository resources create an imperative for an application profile that can enhance discoverability and access of objects. To accomplish this, the project drew elements from existing standards such as Dublin Core (DC) and the Categories for the Description of Works of Arts' new version CDWA Lite. These standards were chosen because they offer important means of describing dataset objects and facilitating their exchange (Zeng & Qin, 2008). In order to identify elements from these standards that are most salient to my project, I evaluated many of the common DC and CDWA Lite elements that are often adopted for data-focused metadata applications. Schema Creation

Having developed appropriate metadata element sets for my project, I created a schema file that would provide a framework for creating .xml documents based on the element specifications I developed. Generally, creating the schema (.xsd) file entailed establishing type properties that would provide the descriptions of what should be contained within the element sets when an .xml document is created based on these element sets. The various types I used included string (xs:string) and dates (xs:date). An important part of creating the schema was deciding on the cardinality of element sets. In other words, the number of times an element could be repeated in the document, and whether they ought to be mandatory or optional. For instance, I reasoned that an element like title may appear only once in a document since dataset objects or resources usually have single titles. However, elements such as keyword> could appear multiple times since there are often more than one keyword for describing datasets. Conclusion

The importance of data exchange and resuse for research within and outside academia cannot be overstated. In the absence of this phenomenon, research works may not be easily validated or replicated, which could have important repurcussions on the advancement of knowledge. Data repositories have emerged to make sharing and accessing datasets among researchers both easy and convinient. Efforts to improve description of metadata for enhanced repository objects discovery should therefore be a priority among information and research professionals. Completing this project and the (Metadata) course has taught me an important lesson about best practices for creating and describing information resources for enhanced discoverability and access
